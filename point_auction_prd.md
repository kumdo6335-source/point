# 포인트 경매 (Point Auction) PRD — v1.0 (확정)

작성일: 2026-07-15
구현 환경: Antigravity / 기존 Firebase 프로젝트 사용

---

## 1. 개요

- **제품명**: 포인트 경매 (가칭)
- **한 줄 정의**: 모둠별로 100포인트를 가지고 4×4 16개 땅을 비밀 입찰 경매로 획득, 빙고 1줄 완성 또는 최다 땅 확보로 승리하는 실시간 교실 게임 웹앱
- **사용 맥락**: 교사 화면 TV 미러링 + 모둠당 태블릿 1대(가로), 25명 내외 학급, 40분 수업 내 운영
- **데이터 정책**: 게임 종료 시 즉시 삭제 + TTL 자동 정리 안전망 (5장 참조)

## 2. 사용자·환경

| 역할 | 기기 | 화면 특성 |
|---|---|---|
| 교사(호스트) | PC → TV 미러링 | 원거리 가독성, 진행 제어 중심 |
| 모둠(학생) | 모둠당 태블릿 1대, 가로 | 큰 터치 타깃, 한 화면 한 작업 |

- 모둠 수: 방 생성 시 2~8 설정, 기본 6
- 모둠명: A~H 고정 알파벳 (설정한 모둠 수만큼 활성화)
- 입장: 방코드 입력 → 활성 모둠 중 선택 (이미 선점된 모둠은 선택 불가)
- 인증 없음, 방코드 비밀 유지로 접근 통제

## 3. 게임 규칙 (확정)

### 3.1 기본 구조

- 모든 모둠 시작 포인트: 100
- 1번 → 16번 땅 순서대로 경매 진행
- 입찰 방식: **비밀 동시 입찰** — 다른 모둠의 입찰액·잔여 포인트를 볼 수 없음
- 0포인트 제출 = 미투자

### 3.2 낙찰 판정

1. 단독 최고액 모둠이 낙찰
2. 최고액 동점 시 해당 모둠들 모두 탈락, 그 다음 금액으로 내려가 단독 모둠 탐색 (연쇄 적용)
3. 어느 금액대에도 단독 모둠이 없거나 전원 0포인트면 **유찰** (땅 주인 없음)
4. 투자한 포인트는 낙찰 여부와 무관하게 **전액 소멸** (환불 없음)

### 3.3 종료·승리 조건

- 어느 모둠이든 **빙고 1줄(가로 4·세로 4·대각선 2, 총 10줄) 완성 즉시 게임 종료·해당 모둠 승리**
- 16번 땅까지 빙고 없이 종료 시: 최다 땅 모둠 승리, 동수면 **공동 우승**

### 3.4 정보 공개 정책

| 정보 | 경매 중 | 낙찰 직후 | 게임 종료 후 |
|---|---|---|---|
| 각 모둠 입찰액 | 비공개 | 비공개 | **공개** (땅 클릭 시 모둠별 입찰액) |
| 낙찰 모둠 | — | 공개 (모둠명만, 금액 비공개) | 공개 |
| 잔여 포인트 | 자기 모둠만 | 자기 모둠만 | 공개 (가정) |
| 제출 완료 여부 | 교사 화면에만 표시 (금액 제외) | — | — |

- 포인트 소진 모둠을 위한 구제 장치 없음 — "누가 포인트가 없는지 모른다"는 정보 비대칭 자체가 전략 요소

### 3.5 규칙 팝업

게임 시작 시 교사·학생 화면 모두 표시. 포함 내용:

- 시작 포인트 100
- 1~16번 땅 순서 경매, 비밀 입찰
- 투자 포인트는 낙찰 여부와 무관하게 미환불
- 승리 조건: 빙고 1줄(가로·세로·대각선) 즉시 승리 / 없으면 최다 땅, 동수면 공동 우승
- **동점 연쇄 규칙 그림 예시 1개** (예: A모둠 30·B모둠 30·C모둠 20 → A·B 동점 탈락, C가 20으로 낙찰)

## 4. 화면·흐름

### 4.1 교사 화면 (TV 기준)

1. **방 생성**: 모둠 수 설정 → 방코드 생성·대형 표시
2. **대기실**: 모둠별 입장 현황 → "게임 시작" 버튼
3. **규칙 팝업**: 시작 직후 전체 규칙 표시, 닫으면 1번 땅 경매 개시
4. **경매 진행 화면**:
   - 4×4 보드 — 현재 경매 땅 강조, **낙찰 땅은 모둠 색 배경 + 해당 모둠 알파벳이 칸을 크게 채움** (TV 원거리에서도 식별 가능)
   - 모둠별 제출 완료/미제출 표시 (입찰액 미표시)
   - "강제 마감" 버튼 — 미제출 모둠은 0포인트 처리
   - 전원 제출 시 자동 마감
5. **결과 공개**: "N번 땅 → B모둠 낙찰" 또는 "유찰" 표시 (금액 비공개) → "다음 땅" 버튼으로 진행
6. **되돌리기**: 직전 경매 1회 undo (배정 취소·포인트 복원·해당 땅 재경매)
7. **종료 화면**:
   - 승리 모둠·사유(빙고 / 최다 땅 / 공동 우승) 표시
   - 보드에서 땅 클릭 → 해당 땅의 모둠별 입찰액 상세 공개
   - "게임 종료·데이터 삭제" 동작 포함 — 종료 확정 시 방 데이터 즉시 삭제

### 4.2 학생(모둠) 화면 (태블릿 가로)

1. 방코드 입력 → 모둠 선택
2. 규칙 팝업 확인 후 대기
3. **입찰 화면**:
   - 현재 땅 번호, 4×4 보드(소유 현황 — 교사 화면과 동일하게 모둠 색 + 알파벳 표기), 자기 모둠 잔여 포인트
   - 입찰액 입력(숫자 키패드) + 제출 버튼
   - 잔여 포인트 초과 입력 시 즉시 오류 표시·제출 불가
   - 제출 후 수정 불가, "제출 완료 — 다른 모둠 대기 중" 표시 (가정: 제출 후 수정 불가)
4. **결과 화면**: 낙찰 모둠(또는 유찰)만 표시
5. **종료 화면**: 교사 화면과 동일한 결과·입찰 내역 열람
- 새로고침·이탈 시 같은 방코드 + 같은 모둠 재선택으로 진행 상태 복원

## 5. 기술 사양

- **스택**: React + Firebase **Firestore** (`onSnapshot` 실시간 구독) — 스트림스와 동일 구조, Antigravity에서 구현
- **낙찰 판정·땅 배정**: Firestore 트랜잭션으로 원자 처리 (전원 제출 확인 → 판정 → 배정 → 포인트 차감)

### 5.1 Firestore 컬렉션 구조

```
rooms/{roomCode}
  ├ config: { teamCount, status: waiting|playing|finished, currentLand: 1~16, expireAt }
  ├ teams/{A~H}: { points, ownedLands: [], connected, expireAt }
  ├ lands/{1~16}: { owner: teamId|null, bids: { A: 30, B: 0, ... }, status, expireAt }
  └ history: [ 직전 경매 스냅샷 (undo용 1개) ]
```

### 5.2 데이터 삭제 정책 (이중 장치)

1. **즉시 삭제**: 교사가 종료 화면에서 게임 종료 확정 시 방 문서 전체(하위 컬렉션 포함) 즉시 삭제
2. **안전망(TTL)**: 모든 문서에 `expireAt`(생성 +24h) 필드 + Firestore TTL 정책 — 교사가 종료하지 않고 이탈해도 자동 정리 (TTL 특성상 만료 후 최대 72h 내 삭제)
3. Firebase 콘솔에서 TTL 정책 1회 설정 필요 → 콘솔 설정 체크리스트에 포함

### 5.3 Firebase 콘솔 설정 체크리스트

- [ ] Firestore 데이터베이스 활성화 (기존 프로젝트 사용)
- [ ] 보안 규칙: 방코드 기반 읽기/쓰기 허용 (인증 없음 전제)
- [ ] TTL 정책 설정: `expireAt` 필드 기준, rooms 및 하위 컬렉션 그룹 대상

## 6. 구현 단계

1. 방 생성·입장·모둠 선택
2. 경매 사이클 (입찰 제출 → 자동/강제 마감 → 낙찰 판정 → 결과 공개 → 다음 땅)
3. 빙고 판정·종료 처리
4. undo·재접속 복원
5. 종료 화면 입찰 내역 공개·데이터 삭제·스타일링

## 7. 리스크·검토

- **40분 운영**: 16회 경매 × (입찰 상의 1~2분 + 결과 확인) ≈ 25~35분. 규칙 설명 포함 시 빠듯할 수 있음 — 교사 강제마감으로 템포 조절 가능. 필요 시 모둠 수 축소 또는 첫 게임은 1~4번 땅만 연습판으로 진행하는 운영 팁을 교사 안내 문서에 포함 권장 (규칙 팝업에는 미포함)
- **초반 과투자 후 이탈**: 구제 장치 없음(확정). 관전 상태에서도 보드·낙찰 발표는 계속 보이므로 최소한의 관여는 유지됨
- **동점 연쇄 규칙의 이해 난도**: 규칙 팝업에 그림 예시 1개 포함으로 대응 (3.5 참조)
